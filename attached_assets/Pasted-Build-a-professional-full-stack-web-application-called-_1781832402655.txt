BACKEND API ROUTES (organize these inside routes/email.ts and routes/share.ts):

POST /api/generate-emails
- Receives: intent, keyFacts, tone, businessName
- Calls Groq API with both models simultaneously using Promise.all
- Model A (llama-3.3-70b-versatile) uses an ADVANCED prompting technique: a system message establishing an expert email-writer persona, PLUS two full few-shot example pairs presented as real conversation turns (system → user → assistant → user → assistant → user) before the actual request
- Model B (openai/gpt-oss-120b) uses a BASIC prompting technique: a single plain-instruction user message only, no system prompt, no examples
- Returns both emails as { modelA, modelB }

POST /api/evaluate
- Receives: emailText, keyFacts, tone
- Metric 1 (Fact Recall Score, 0-10): pure JavaScript keyword matching — split key facts into lines, strip common stop words, check if at least 50% of each fact's meaningful words appear in the email, score = (facts matched / total facts) * 10
- Metric 2 (Tone Accuracy Score, 0-10): LLM-as-judge — call llama-3.3-70b-versatile with a structured 0-10 rubric, instruct it to respond with ONLY a JSON object like {"score": 8} for reliable parsing
- Metric 3 (Professional Quality Score, 0-10): LLM-as-judge — same approach as Metric 2, judging grammar, structure, and clarity
- Overall score = average of all three, rounded to 1 decimal
- Returns { factRecallScore, toneAccuracyScore, professionalQualityScore, overallScore }

POST /api/share
- Stores the submitted comparison data in an in-memory Map with a randomly generated short ID
- Returns { id }

GET /api/share/:id
- Looks up the ID in the in-memory Map
- Returns the stored data, or a 404 JSON error if not found
